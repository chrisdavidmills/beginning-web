Welcome to It's my web. This is a ten week course, aimed at teaching complete beginners some basic web design skills. We won't have anywhere near enough time to make you masters at building websites, but we will be able to get you started, and give you some enthusiasm to go off on your own and learn more.

In this session we will get started by looking at why the Web is interesting and how it works, and then hacking around with your school website near the end!

What the web is, and why it is so interesting

You can also view "What is the Web?" on its own web page. This presentation was created using the rather excellent slid.es, which is powered by the RevealJS JavaScript library.

How does the Web work?

The Web is built on a series of technologies for information transfer, plus some programming languages that you use to specify how you want that information to look. A simplified view follows:

  • The client: This is your computer at home, which you use to access the Web
  • The server: This is a computer sat in a room somewhere, which stores websites. When you access a website on your computer, a copy of the website data is sent from the server to your client machine for you to look at.
  • Your Internet connection: Allows you to send and receive data on the Web. It's basically like a giant street that cars and people can travel down.
  • TCP/IP: Defines how your data should travel down that road. This is like the cars and buses people use to get places.
  • HTTP: Defines how and when data should be sent between the client and the server. This is like some people deciding what journeys they need to go on down the road, how far they need to travel, what mode of transport they need to use, etc.
  • DNS: Domain Name Servers are like an address book for Websites. When you type in a web address in your browser, before the website is retrieved the browser goes to the DNS to find out where it lives, like when you look up someone's address so you can go and visit them. Try typing http://212.58.251.195 into your web browser.
  • HTML, CSS and JavaScript: The main three programming languages that websites are built from.
  • Assets: This is a collective noun for all the other stuff that makes up a website, such as image files, MP3s and videos, Word documents, PDFs, etc.

A game of client/server

Let's play a game! To play this we need:

  • Some lego bricks (or some coloured cards of some kind)
  • One person to play a web server
  • One person to play HTTP
  • 2-3 people to play clients
  • One person to play DNS
  • and a decent number of people to play our website data.

First, a typical successful web request:

  1. First the clients request a web page
  2. The client web browsers look up the IP address of the website using the DNS
  3. HTTP informs the server that the clients are requesting a website, with a message sent to the server. This is called an HTTP request.
  4. There is no reason why the clients can't have this website, so the server responds with "200, OK". HTTP gives this response back to the client.
  5. The website data is sent down the pipe to the clients. A copy is sent to each.
  6. The clients assemble the data into a working website that it's users can interact with.

Now, an unsuccessful web request:

  1. First the clients request a web page, but they've got the address slightly wrong.
  2. The client web browsers look up the IP address they've been given using the DNS
  3. The website cannot be found, so the HTTP response "404, NOT FOUND" is given back to the client.

When data is sent down the pipe, it is broken up into little tiny bits called packets. Each one of our data people really represents a single packet! If the data were not broken into packets before being sent to the client computers, it would be much harder to send around. It's the same principle as getting a wardrobe in through your front door, up your stairs and into your bedroom. This is a lot easier to do if you carry the wardrobe upstairs in pieces and then assemble it in the bedroom, rather than carrying it upstairs already assembled!

The open source, hackable web

As I mentioned before, one of the main principles the Web was created on was being developed openly and made available to all for free. This is a very good thing for humanity: think about how much the Web shapes our lives. It would be very scary indeed if one company or government was in control of the Web, and started developing it in secret.

The Web is built on a series of open standards, meaning that your code can run on any computer that has a web browser, and you don't have to pay anything to create websites, beyond the cost of the computer you are using, and an internet connection.

Standards are important - they make our lives much easier. It is the same principle as standards in the real world. Think of how convenient it is that all appliances in our country use the same power sockets, and all screws are screwable using one of about 2-3 types of screwdriver.

Another aspect of this openness is that you can view the code of any website, and see how they work. Every web browser has a "View source" option. Let's try it. This is very helpful for learning. The web development community is also very open, collaborative and supportive. Lots of people publish articles to teach you how to do web design and development. For example, I publish most of my work on the Mozilla Developer Network. Literally hundreds of people contribute to this site.

You can even write code to affect the way you view and use other people's websites. These tend to take the form of browser addons or extensions. Let's look at something similar.

Let's look around the web, with Xray goggles!

We'll round off this week with a first look at some web code, via an application called Xray Goggles.

  1. Click on the above link.
  2. On this page, click the X-Ray Goggles button. You'll see that now as you move your mouse pointer around the page, the different parts of the page highlight to tell you what code is underneath each bit.
  3. Click the "X-Ray Goggles" title at the top of the page: A window will open up allowing you to edit the page!
  4. Change the text of the title to something else
  5. Click the "Commit changes" button: you'll now see that the page has been changed
  6. You can also press delete/backspace to delete bits of the webpage, and P to publish your hacked version of the page!
  7. Play with the page for a few minutes, then turn Xray Goggles off by pressing Esc.
  8. You can make Xray Goggles available on any page by putting it on your browser: You can drag the Xray Goggles button onto your bookmarks toolbar, or add it to your bookmarks menu.

Note: You have only changed your own copy of the web page, not the original on the server. If you reload the page (Ctrl + R or Cmd + R on Mac) you'll see that the page returns to its original state.

Hacking the school website

Let's load up the school website and have a go at changing that. Once you've finished, you can publish your remixed version of the web page at it's own web address by pressing P.

If you want to update an image on a web page, it is more complex, but you can still do it without too much trouble. First lets look for an image to insert on your page. The best way to find images to insert is to use Google image search, which is a Google site that allows you to search just for images.

  1. Open Google Images in a new tab.
  2. Search for something you'd like to display, and click on an image result. The image should come up big and highlighted.
  3. Click on the View image button; this will bring just that image up in the browser.
  4. Select the whole web address of the image that you should now see in the address bar of your browser and copy it (you can also do this in Firefox by Right/Ctrl + clicking and selecting Copy image location; most other browsers have a similar option.)
  5. Go back to the tab you are running Xray Goggles on.
  6. Click on one of the images: it should come up with the Xray Goggles window, showing an <img> tag, with a bunch of other information inside it. If you don't see this, press Cancel and try something else. Not all images are as they seem.
  7. The bit you need to change is the src attribute, which should look like src="http://some.kind-of-web-address.com". You need to replace the bit inside the quote marks (" ... ") with the web address you copied from Google Images earlier.
  8. Press Save, and you should see the image updated.

You can find more information and ideas about searching for images in Lesson 11: Supplementary techniques.
