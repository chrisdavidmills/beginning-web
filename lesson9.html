Hey there! In this section we will again explore a more advanced topic — positioning — and then take some free time out to do some more work on our projects.

Positioning

Positioning is a CSS feature that you can use to make more interesting CSS layouts. The idea is that you take the default position that an element is laid out in — how it normally appears on the page — and then modify its position so that it ends up appearing somewhere else, with some interesting results.

Without further ado, let's look at an example!

to start with:

  1. Create a new Thimble page.
  2. Add a style element to the HTML head.
  3. Add the following two div elements to the HTML body
    4. <div class="first"></div>
<div class="second"></div>
  4. Add the following CSS to your page:
    5. div {
  font-size: 200%;
  line-height: 150px;
  color: rgba(0,0,0,0.3);
  width: 250px;
  height: 150px;
  margin: 0 auto;
  text-align: center;
}

.first {
  background-color: rgba(255,0,0,0.5);
  position: ;
}

.second {
  background-color: rgba(255,0,255,0.5);
}

At this point we have two containers, the same size, but different colours.

I've made them slightly transparent so we can see both elements even as one moves over the other. The transparent colours are created using these rgba() blocks. To find out how they work, read the MDN color value page.

You'll notice that in the rule with the .first selector, we have a blank position property. Lets fill it in with different values in subsequent sections to show what options are available.

Static positioning

To start off, fill in a value of static in the position property. This is the default position type, so you'll see no difference. Moving on!

Relative positioning

Now change the value of static from static to relative. We now have the ability to move the element relative to its original laid out position on the page. We specify the amount we want to move the element using the top, bottom, left and right properties. Add these lines below the position property:

top: 30px;
bottom: ;
left: 30px;
right: ;

You'll now see that the top div has moved down and to the right. Try experimenting with adding other values for a few minutes. You might think it is weird that the values seem to move the element in the opposite direction to what you might expect, but think of it as you specifying the side from which you are pushing to get the element to move.

Now let's set all the top, left, etc. values to 0 (or just blank out the values), in preparation for the next part.

Absolute positioning

Absolute positioning is quite different. This removes the element from the document layout completely so that it no longer takes up any space and you can position it anywhere inside the body element you like. It works a bit differently to relative, so lets explore it.

Change the position value from relative to absolute. You'll now see that the first div moves up to the top left corner of the Preview area, and sits over the top of the second div. The second div is now positioned as if the first div didn't exist at all!

Now try setting the top and right property values both to 50px. The element will now move so that it is 50 pixels away from the top of the body, and 50 pixels away from the right of the body. This is markedly different behaviour to relative position.

Last in this section, I'd like you to go again to the Lorem Ipsum generator. Generate about 5 or 6 paragraphs of it, paste them into your HTML, and put all the paragraphs inside p elements. Basically, put in enough to make the page scroll. Note how the absolutely positioned element stays in the same place relative to the body element, and scrolls as the page scrolls.

Fixed positioning

Next, change the value of the position property from absolute to fixed. Let's explore the difference — try scrolling the page now!

Why does this occur? This is because absolute positioning generally positions an element relative to the body element's perimeter (I say generally, as this can also be changed). Fixed positioning instead positions elements relative to the browser window, which is why the element stays in the same position when the window is scrolled. You can have a lot of fun with this, for example if you want to have one important element always staying in view, like a map, or a menu.
