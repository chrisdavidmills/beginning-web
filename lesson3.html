This week we will start to learn about HTML, the language that is used on the Web to structure all content, and give it meaning (by this we mean "is it a paragraph?", "is it an image?", etc.) To start off with, we'll look at some real basics and build a few isolated examples. Then we'll move on to our project, and add some new content.

Before we start, open up a new Mozilla Thimble page; we'll use this for some of our work below.

You can find a useful reference to the most common HTML features you'll want to use: see the HTML Cheatsheet.

What does HTML look like?

HTML is composed of tags, which wrap around bits of content that you want to do something to. This is the same principle used in programs you'll have already used, like Microsoft Word, or Apple Pages. When you select a bit of text and press the bold button to turn it bold, what you're actually doing is tagging that text with bold tags ... it is just that in those kinds of programs you don't see the code; it all happens in the background.

The proper name for tagging in HTML is marking up. All of the content you're reading now is marked up in HTML.

Let's have a look at a typical, simple HTML example:

<p>A simple paragraph of text.</p>
  • The < followed by the tag name followed by the > is called the opening tag.
  • The < followed by the forward slash (/) and tag name followed by the > is called the closing tag.
  • The stuff in between the opening and closing tags is called the content.
  • The whole lot together is called an element.

Try entering this paragraph element into your Thimble page: anywhere in between the opening and closing <body> ... </body> will do.

Before we move on, let's talk about attributes. Attributes are extra bits of information that you can add to an HTML element to do extra things to it. For example:

<p style="color: blue;">A simple paragraph of text.</p>

Here we are adding a style attribute to our p element. Inside this, we can insert CSS to add styling just to this one element. In this case, we've just turned the text blue. There are lots of different attributes and CSS styles you could use, but we'll leave this there for now.

What does an HTML document look like?

An HTML document is generally split up into two parts: the head and the body. There are a couple of extra bits that don't quite fit into either, but these are fairly easy to understand too. Let's have a look at all the different parts:

<!DOCTYPE HTML>
<html lang="en-US">
<head>
  <meta charset="UTF-8">
  <title>It's my Web: Lesson 4</title>
  <link rel="stylesheet" type="text/css" href="style.css"> 
</head>
<body>
  <h1>This is my web page.</h1>
</body>
</html>

In this example you can hover over the different lines with the mouse pointer to get a description of what they are, or even cycle through the different line descriptions by pressing the tab key (or whatever other keyboard controls your browser has for cycling through focusable elements).

  1. The doctype is not an element (technically, it's called a processing instruction.) It basically points to a ruleset that defines rules how the HTML in the document should be written. You don't really need to know anything else about this, at least not for now. This always goes at the top of your HTML document.
  2. The html element wraps around the entire HTML document, except for the doctype. You should always include this second in an HTML page. This is sometimes called the root element of the page.
  3. The head of the document is wrapped by the head element. This contains all the information that describes your document, and does useful things to it, but which doesn't actually appear on the page. Here we've got:
    • A meta element, which defines what letters, numbers and other characters are allowed in the document. utf-8 basically means "every character available in the entire world." There is no reason to not use this.
    • A title element, which controls the text that appears in the web browser tab when this page is loaded up (and the text used when you bookmark the site.) You can change this to whatever you like.
    • A link element, which references a CSS file. This is just one way in which CSS styles can be applied to HTML (we already saw another one, earlier on, can you remember what it is?)
  4. The body of the document is wrapped by the body element; this is where all your web page content goes â€” the stuff you want to show in the web browser. At the moment I've just put a single level 1 heading there, for example's sake.

You'll notice that there are a couple of elements here that only have a single tag, rather than a opening and a closing tag. They are called empty elements, because they have no content inside them. You'll meet a few of these throughout the course.

Headings and paragraphs

Headings and paragraphs make up a lot of web content. Just as you would in Microsoft Word or another word processor, you create a document by having a single top level heading. then subsections are created using second level headings, then sub-subsections are created with third level headings, etc. In HTML this is done using heading levels h1, h2 and h3 (You can go right the way down to h6, but you really shouldn't be using this many levels of headings). Any content can be placed in between the headings, but for now, lets stick to paragraphs. Let's go back to a previous example.

<h1>My website is all about food</h1>

<p>I <strong>love</strong> food. It is what I look forward to
<em>every</em> morning, lunchtime and evening. My favourite foods are
 as follows:</p>

<h2>Cheese</h2>

<p>Let's start with cheese. Cheeses are like an international culture 
trip, with every country having its own famous varieties.</p>

<h3>Edam</h3>

<p>Edam is...</p>

<h3>Stilton</h3>

<p>Stilton is...</p>

<h3>Gorgonzola</h3>

<p>Gorgonzola is...</p>

So here we have a top level heading that begins the entire content, then a second level subsection, then the subsection has three sub-subsections marked out by third level headings. Lets try adding some of our own headings and paragraphs to our projects.

Lists

HTML allows us to create lists, which is very significant. Life is full of lists, from obvious lists such as recipes and shopping lists, to less obvious lists. When you play with a toy like lego or a video game, you are making lists - "what are the next five pieces I should put on my model", or "in what order do I need to press the XBox 360 buttons to do the ultra combo?" In this section I'll introduce you to two different types of list.

Unordered lists are lists where the order of the list items doesn't matter. For example, a short shopping list:

  • Eggs
  • Milk
  • Cheese
  • Co-Co Pops
  • Orange Juice

We'd mark this up like so:

<ul>
  <li>Eggs</li>
  <li>Milk</li>
  <li>Cheese</li>
  <li>Co-Co Pops</li>
  <li>Orange Juice</li>
</ul>

The opening <ul> tag marks where the list starts, the closing </ul> tag marks where the list ends, and the list items are all wrapped in a pair of <li> ... </li> tags.

Ordered lists are lists where the order of the items does matter. For example in a recipe, you would not successfully make the dish if you followed the instructions out of order. The ordered list is exactly the same as the unordered list, except that the container element is ol, not ul.

<ol>
  <li>Preheat the oven to 150 C / gas mark 2.</li>
  <li>Add butter and sugar in a bowl and mix well until 
    light and fluffy.</li>
  <li>Add the vanilla, mix, then add the flour and mix well.</li>
  <li>Roll out to about 5mm. Cut into shapes.</li>
  <li>Bake for 25 minutes or until golden brown.</li>
</ol>

Emphasis

HTML also has some elements that allow us to make some words stand out, or be more important than others. Here we'll look at two:

  • strong is for more important words
  • em is for emphasis, on words that you might say differently

Let's look at a couple of examples:

<p>I have been away from home for <em>far too long</em>...</p>

<p>When you phone up, be sure to <strong>quote your 
reference number</strong>.</p>

You can nest both if you like, so that you get emphasised and important:

<p>When you phone up, be sure to <strong>quote your 
reference number: <em>3387</em></strong>.</p>

When you nest elements, you need to do them correctly, so you must close elements in the opposite order that you opened them in. The following is wrong:

<p>When you phone up, be sure to <strong>quote your 
reference number: <em>3387</strong></em>.</p>

Block versus inline elements

As a brief note, when you start to do more web design you will hear the terms inline element and block level element. A block level element takes up space on the page, and appears on a new line below the previous content. Headings and paragraphs are good examples. An inline element is always put inside a block level element, and does not take up space on the page as such. It does not appear on a new line. emphasis and strong are perfect examples. You can put block level elements inside other block level elements, but you can't put block level elements inside inline elements.
