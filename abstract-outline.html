Outline for St. Chad's web basics course

Chris Mills, October 2013

The following document provides scope and basic structure for a ten hour (one hour per week) course to be taught to 10-11 year olds at St Chad's school. This is a pilot course, aimed at gauging interest in technology/programming/web development in primary school children and hopefully leading to more structured learning and curricula of this type in the Saddleworth area.

Title: It's my Web! (Understanding how the web works, and how to change it)

Scope

The Web is a good place to begin learning about how technology works, and fundamental ICT concepts such as programming. It is ubiquitous and familiar, easy to learn compared to other programming platforms, and there is a lot of help available due to the web's largely free and open nature. This course will aim to teach children some background information on how the web works and why the web is so interesting, the basic fundamentals of HTML (the language used to structure data on the Web), CSS basics (the language used to style and layout the Web), possibly then moving on to some JavaScript (the web's main programming/logic language) if there is time.

There certainly isn't time in 10 hours to teach anything in any great depth, but here we will aim to give children what they need to produce a simple web page that can be presented via the school web site. Because the ability level of the children is likely to vary quite a bit, we will provide some advanced questions/exercises for any children that are finding the basic course level too easy and want to stretch themselves.

Structure

What follows is a course structure, broken up week by week.

Week 1

  • Short presentation: what the web is, and why it is so interesting
  • How does the web work? Taught using interactive games (cards, lego?)
    • Basic client/server interactions
    • HTTP
    • Client-side versus server-side technologies
    • Web standards - HTML, CSS and JavaScript
  • Open source, view source, hackable web
  • A look round the web, using Xray goggles. Lets hack the St. Chad's web site! (ok, not permanently.)

Week 2

  • Using Mozilla's Thimble tool to write some simple web pages
  • Getting every child logged in (do we need to create official school accounts for this purpose?)
  • A tour of the tool
  • Remixing other people's makes.
  • Choosing a project to build (I'll provide a few templates to get started from, e.g. diary page, band fan site, newsletter, holiday journal. Any others to suggest that might help with their schoolwork?)

Week 3

  • Getting started with HTML
  • The anatomy of an HTML element (compare markup to systems they'll have already used, like Microsoft word, or PDFs)
  • HTML head and body (what are they for, what do they contain?)
  • Page title
  • Headings and paragraphs
  • Lists
  • emphasis, quotations
  • Block versus inline elements

Week 4

  • HTML Images
    • Finding images to use (should I bother mentioning copyright at this stage?)
    • Including an image in your page
    • Sizing
    • Titles and alt text
    • Note on web accessibility, show youtube screen reader video
  • HTML Links
    • Linking to other resources on the web (even images!)
    • Adding links to your page
  • Other stuff in HTML that I should mention for completeness, but which are boring: forms, tables

Week 5

  • Starting on CSS
  • Anatomy of a CSS ruleset
  • Where does the CSS go? (talk mainly about internal, but also explain external and inline)
  • Element selectors
  • Other types of selectors
  • Some basic properties to play with: color, background-color, font-size, padding, border, box-shadow, text-shadow, border-radius, linear gradient (advanced)

Week 6

  • Width and height (max- and min- ?)
  • The box model
    • How is HTML laid out on a page?
    • Content, margin, padding, borders
    • Margin collapse
  • Background images

Week 7

  • CSS layout
  • Floats
  • 2 column layout
  • Positioning (for advanced students)

Week 8

  • Free week for carrying on with project?
  • Optional advanced topics for really advanced students (media queries, animation, link states)

Week 9

  • Free week for carrying on with project?
  • Optional advanced topics for really advanced students (JavaScript basics?)

Week 10

  • How do you become a web developer?
  • You don't need a degree, you just need interest and passion
  • What tools do you need?
    • Web browsers
    • Code editor
    • Graphics editor
    • Server space
    • A file structure
